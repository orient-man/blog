## Context

Post listing pages currently display 160-character excerpts auto-generated by `generateExcerpt()` in `src/lib/utils.ts`.
The original WordPress blog rendered full post content on listing pages, giving readers substantial context.
The current 160-character limit produces minimal snippets that lose the author's writing style.

The RSS feed script (`scripts/generate-feeds.ts`) uses an independent excerpt implementation capped at 200 characters.
SEO metadata on post detail pages reuses the same 160-character excerpt.

## Goals / Non-Goals

**Goals:**
- Increase excerpt length to ~500 characters to surface roughly one paragraph of content.
- Unify excerpt length across the main app and RSS feed generation.
- Ensure PostCard layout accommodates longer excerpts without visual breakage.

**Non-Goals:**
- Changing the PostCard visual design or layout structure.
- Adding a "read more" link or explicit truncation indicator beyond the existing ellipsis.
- Supporting per-post custom excerpt lengths via frontmatter (the existing `excerpt` frontmatter override remains but is not extended).
- Rewriting the RSS feed to share `generateExcerpt()` (the feed script keeps its own stripping logic; only the length changes).

## Decisions

### D1: Excerpt length of 500 characters

**Choice:** Increase default from 160 to 500.

**Rationale:** 500 characters is roughly one full paragraph — enough to convey topic and tone without overwhelming the listing page.
The original WordPress blog showed entire posts; 500 chars is a pragmatic middle ground that preserves character without infinite scroll.

**Alternatives considered:**
- 300 characters — still too short to preserve a meaningful paragraph.
- Full content on listing pages (matching WordPress exactly) — would make the listing page very long and hurt scanability.
- Configurable per-post via frontmatter — unnecessary complexity for a single-author blog (Constitution Principle I: Simplicity).

### D2: Adjust CSS line-clamp from 6 to 10

**Choice:** Increase the Tailwind `line-clamp-6` on PostCard's excerpt `<p>` to `line-clamp-10`.

**Rationale:** At `text-sm` (14px) with `leading-relaxed` (1.625 line-height), 500 characters typically wrap to 6-8 lines.
Bumping the clamp to 10 provides headroom so the CSS safety net does not inadvertently re-truncate text.

**Alternatives considered:**
- Remove line-clamp entirely — risk of extremely long excerpts from edge-case content.
- No change (keep 6) — would silently clip many 500-char excerpts, defeating the purpose.

### D3: Keep RSS feed stripping logic independent

**Choice:** Only change the length constant in `generate-feeds.ts`; do not refactor to share `generateExcerpt()`.

**Rationale:** The feed script runs as a standalone prebuild step.
Sharing code would require extracting a common module or making the script import from `src/lib/`, adding build complexity for minimal benefit (Constitution Principle I).

## Risks / Trade-offs

- **[Slightly longer listing pages]** Longer excerpts mean more vertical space per card.
  Mitigation: 500 chars is bounded and `line-clamp-10` prevents runaway expansion.
- **[SEO description length]** Google typically displays ~155 characters in search results.
  Longer `og:description` is harmless — search engines truncate on their end. Social media previews (Twitter, Facebook) also handle truncation gracefully.
- **[Edge cases in stripping]** Some posts with heavy MDX/code content may produce low-quality excerpts even at 500 chars.
  Mitigation: Authors can always add an explicit `excerpt` field in frontmatter for any problematic post.
