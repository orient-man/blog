## ADDED Requirements

### Requirement: FR-019 Custom 404 page with adventure-game scene

The blog SHALL have a custom 404 (Not Found) page at `src/app/not-found.tsx` that displays a pixel-art adventure-game scene.
The scene SHALL evoke classic point-and-click adventure games (LucasArts/SCUMM era).
The page MUST include a clear message that the requested page was not found and a link to return to the homepage.

#### Scenario: 404 page renders on missing routes
- **WHEN** the user navigates to a URL that does not match any route
- **THEN** the custom 404 page SHALL render with the adventure-game scene

#### Scenario: Navigation back to homepage
- **WHEN** the user views the 404 page
- **THEN** a visible link or button to the homepage SHALL be present

### Requirement: FR-020 Pixel art assets generated by PixelLab AI

The 404 page pixel art assets (character, trees, moon, environment) SHALL be generated using the PixelLab AI MCP tool.
Generated assets SHALL be stored in `public/images/404/` as static PNG files.
The assets MUST be optimized for web delivery (reasonable file size, appropriate dimensions).

#### Scenario: Assets stored as static files
- **WHEN** the 404 page loads
- **THEN** all pixel art assets SHALL be served from `public/images/404/` as static PNGs with no runtime API calls

### Requirement: FR-021 404 page adventure-game interaction text

The 404 page SHALL include adventure-game-style flavor text (e.g., "You are in a dark forest. The path you followed has ended." or similar).
The text SHALL be rendered in Pixelify Sans to match the retro theme.
The text MUST NOT replace or obscure the functional "page not found" message and homepage link.

#### Scenario: Flavor text displayed
- **WHEN** the user views the 404 page
- **THEN** adventure-game-style descriptive text SHALL be visible alongside the functional navigation elements

### Requirement: FR-022 404 page responsive layout

The 404 page layout SHALL adapt to mobile, tablet, and desktop viewports.
On smaller screens, the pixel art scene MAY be simplified or scaled, but the "not found" message and homepage link MUST remain visible and usable.

#### Scenario: Mobile viewport usability
- **WHEN** the user views the 404 page on a viewport under 640px wide
- **THEN** the "not found" message and homepage link SHALL be visible without horizontal scrolling
